# Implementation Logs

One folder per feature, numbered and named to match [`../features.md`](../features.md).

Each feature folder contains:

| File / Folder     | Purpose |
|-------------------|---------|
| `tests/`          | Prototype-grade tests that validate the feature works, has no functional gaps, and honours its documented constraints. |
| `test_review.md`  | Results and observations from running those tests. |
| `changes.md`      | Modifications made during development, especially deviations from the original plan. |
| `errors.md`       | Errors encountered and how each was resolved. |

Tests here are intentionally **prototype-grade** — enough to prove the feature behaves as described, not production coverage.
